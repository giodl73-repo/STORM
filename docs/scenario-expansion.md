# STORM Scenario Expansion

This expansion sketches exactly 50 product-neutral scenario/capability candidates for
STORM. Each candidate stays inside STORM ownership: weather systems, hazard events,
warning signals, exposure, preparedness tradeoffs, recovery windows, evidence, and
validation. Named sibling repos receive bounded downstream handoffs only.

## Repeated scoring gaps surfaced

- Warning lead time and certainty need to be explicit, especially when warnings are
  false, distant, or repeated.
- Exposure should name the affected asset or population class, not only the hazard.
- Preparedness should create a visible tradeoff in labor, material, route access,
  water, fuel, legitimacy, or trust.
- Recovery windows should include the earliest useful action and the risk of a
  second system arriving before recovery finishes.
- Handoffs must name the downstream owner and stop before map effects, economics,
  ecology simulation, ritual adjudication, identity governance, narrative
  interpretation, or game mechanics.
