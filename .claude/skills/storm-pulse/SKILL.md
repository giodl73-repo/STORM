# STORM Pulse

Use this skill for one STORM implementation pulse.

## Pulse rules

1. Read `context\waves\PHASES.md`, the active `WAVE.md`, and the target pulse.
2. Keep `storm-core` product-neutral.
3. Preserve bounded downstream handoffs; STORM records hazard pressure but does
   not own maps, economics, ecology, ritual legitimacy, identity, or game rules.
4. Add or update fixtures before broad consumer adoption.
5. Run the validation commands named in the pulse.
